# 활용 가이드

세일즈맵 MCP를 효과적으로 사용하는 실전 워크플로우와 프롬프트 예시입니다.

## 기본 원칙

Claude는 요청을 수행할 때 다음 순서로 동작합니다:

1. `salesmap-list-properties`로 필드 구조 파악
2. 필요한 도구를 선택하여 실행
3. 결과를 자연어로 요약

복잡한 요청도 여러 도구를 조합해서 처리하므로, **자연어로 원하는 결과를 설명**하면 됩니다.

---

## 워크플로우 예시

### 1. 고객 검색 → 딜 생성

신규 영업 기회를 등록하는 흐름입니다.

```
"김철수" 고객 검색해서, 이 고객에게 "클라우드 전환 프로젝트" 딜을 만들어줘.
금액은 5000만원이고 기본 파이프라인의 첫 번째 단계로 설정해줘.
```

Claude가 수행하는 단계:
1. 고객 검색 → 김철수 레코드 확인
2. 파이프라인 ID 조회
3. 딜 생성 (고객 연결 + 파이프라인 + 금액 설정)

### 2. 딜 현황 분석

영업 파이프라인 현황을 파악합니다.

```
이번 분기에 생성된 딜 중 상태가 "In progress"인 것들을 검색하고,
총 금액과 평균 금액을 알려줘.
```

### 3. 파이프라인 병목 분석

딜이 어느 단계에서 오래 머무는지 분석합니다.

```
딜 ID가 xxxx인 딜의 리드타임 분석해줘.
어느 단계에서 가장 오래 걸렸는지 알려줘.
```

### 4. 견적서 생성

딜에 연결된 견적서를 만듭니다.

```
이 딜에 견적서를 만들어줘.
- 이름: "2026년 1분기 견적"
- 상품: 엔터프라이즈 플랜 12개월, 단가 500만원
```

### 5. 고객 데이터 정리

기존 데이터를 정리합니다.

```
회사명에 "(주)"가 포함된 회사들을 검색해서 목록으로 보여줘.
```

### 6. 팀별 담당자 확인

```
CRM 사용자 목록을 보여주고, 각 팀별로 정리해줘.
```

## 프롬프트 팁

### 구체적으로 요청하세요

```
❌ "딜 보여줘"
✅ "이번 달 생성된 딜 중 금액 1000만원 이상인 것 검색해줘"
```

### 여러 작업을 한 번에 요청할 수 있습니다

```
"홍길동 고객을 만들고, 소속 회사를 ABC 주식회사로 설정하고,
'서비스 계약' 딜도 함께 생성해줘."
```

### 분석을 요청하세요

Claude가 데이터를 조회한 후 직접 분석·요약해 줍니다.

```
"지난 분기 Lost된 딜들을 검색해서, 공통 패턴이 있는지 분석해줘."
```

### 링크를 요청하세요

세일즈맵 웹에서 직접 확인하고 싶을 때 URL을 요청할 수 있습니다.

```
"이 딜의 세일즈맵 링크도 알려줘"
```

## 제한사항

| 항목 | 설명 |
|------|------|
| 검색 필터 | 최대 3개 그룹, 그룹당 3개 조건 |
| 일괄 조회 | 한 번에 최대 20개 레코드 |
| 삭제 | 레코드 삭제는 지원하지 않습니다 (세일즈맵 웹에서 직접 삭제) |
| 파일 첨부 | 파일 업로드/다운로드는 지원하지 않습니다 |
| 이메일·활동 | 이메일 발송, 활동 기록 등은 현재 미지원 (추후 지원 예정) |
