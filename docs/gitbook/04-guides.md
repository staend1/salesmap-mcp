# 활용 가이드

세일즈맵 MCP를 활용한 실전 워크플로우와 프롬프트 예시입니다.

## 기본 원칙

AI 어시스턴트는 요청을 수행할 때 다음 순서로 동작합니다:

1. `salesmap-list-properties`로 필드 구조 파악
2. 필요한 도구를 선택하여 실행 (여러 도구 자동 조합)
3. 결과를 자연어로 요약

복잡한 요청도 여러 도구를 조합해서 처리하므로, **자연어로 원하는 결과를 설명**하면 됩니다.

---

## 영업 관리

### 고객 검색 + 딜 생성

신규 영업 기회를 등록하는 가장 기본적인 흐름입니다.

```
"김철수" 고객 검색해서, 이 고객에게 "클라우드 전환 프로젝트" 딜을 만들어줘.
금액은 5000만원이고 기본 파이프라인의 첫 번째 단계로 설정해줘.
```

AI가 수행하는 단계:
1. `search-objects` → "김철수" 고객 검색
2. `get-pipelines` → 파이프라인·단계 ID 확인
3. `create-object` → 딜 생성 (고객 연결 + 파이프라인 + 금액)

### 딜 상태 변경

```
"클라우드 전환 프로젝트" 딜을 Won으로 변경하고 금액을 6000만원으로 수정해줘.
```

### 고객 + 회사 + 딜 한 번에 생성

```
홍길동 고객을 만들고, 소속 회사를 "ABC 주식회사"로 새로 만들어서 연결하고,
"서비스 계약" 딜도 함께 생성해줘. 금액은 3000만원.
```

AI가 순서대로 처리합니다: 회사 생성 → 고객 생성(회사 연결) → 딜 생성(고객+회사 연결).

### 담당자 배정/변경

```
이번 달 생성된 리드 중 담당자가 없는 것들을 검색해서,
"이영희"에게 일괄 배정해줘.
```

> 담당자 필드에 이름을 입력하면 자동으로 UUID로 변환됩니다.

## 데이터 조회·분석

### 딜 현황 분석

```
이번 분기에 생성된 딜 중 상태가 "In progress"인 것들을 검색하고,
총 금액과 평균 금액을 알려줘.
```

### 파이프라인 병목 분석

딜이 어느 단계에서 오래 머무는지 파악합니다.

```
딜 ID가 xxxx인 딜의 리드타임 분석해줘.
어느 단계에서 가장 오래 걸렸는지 알려줘.
```

AI가 `get-lead-time`으로 각 스테이지별 진입·퇴장 시각과 체류 시간을 분석합니다.

### 고객 360도 뷰

한 고객의 전체 맥락을 파악합니다.

```
이 고객의 정보, 연결된 회사와 딜, 최근 활동 이력을 종합해서 정리해줘.
```

AI가 수행하는 단계:
1. `batch-read-objects` → 고객 상세 정보
2. `list-associations` → 연결된 회사, 딜 ID 확인
3. `batch-read-objects` → 회사, 딜 상세 조회
4. `list-engagements` → 최근 이메일, 노트, 미팅 등 활동 이력
5. 종합 요약 제공

### 변경 이력 추적

```
이 딜의 금액이 언제 변경됐는지 이력 보여줘.
```

`list-changelog`로 필드 변경 내역(변경 전/후 값, 변경자, 변경 시점)을 확인합니다.

### 특정 조건 고객 찾기

```
전화번호에 "010"이 포함된 고객들을 검색해줘.
```

```
최근 30일간 연락이 없는 고객 중 진행 중인 딜이 있는 회사 찾아줘.
```

## 견적서

### 견적서 생성

```
이 딜에 견적서를 만들어줘.
- 이름: "2026년 1분기 견적"
- 상품: 엔터프라이즈 플랜 12개월, 단가 500만원
```

### 기존 견적서 확인

```
이 딜에 연결된 견적서 목록 보여줘.
```

## 노트·메모

### 미팅 노트 기록

```
이 고객에게 노트 남겨줘:
"2026-04-16 미팅 — 클라우드 전환 일정 논의. 5월 중 POC 진행 예정.
담당자: 김철수 차장. 예산 확보 완료."
```

### 활동 이력에서 노트 확인

```
이 딜의 최근 활동 이력에서 노트 내용만 정리해줘.
```

`list-engagements`가 노트 본문을 자동 인라인하므로 별도 조회 없이 바로 확인됩니다.

## 데이터 정리

### 조건부 검색 + 목록 추출

```
회사명에 "(주)"가 포함된 회사들을 검색해서 목록으로 보여줘.
```

```
이메일 필드가 비어있는 고객들을 찾아줘.
```

### 필드 일괄 업데이트

```
이 딜 목록의 담당자를 전부 "박지성"으로 변경해줘.
```

> AI가 각 레코드를 순차적으로 업데이트합니다.

### 팀 구성 확인

```
CRM 사용자 목록을 보여주고, 각 팀별로 정리해줘.
```

## 복합 워크플로우

### Lost 딜 패턴 분석

```
지난 분기 Lost된 딜들을 검색해서, 공통 패턴이 있는지 분석해줘.
금액대, 파이프라인 단계, 체류 시간 등을 비교해줘.
```

AI가 검색 → 상세 조회 → 리드타임 분석을 조합하여 종합 리포트를 제공합니다.

### 주간 영업 리포트 생성

```
이번 주에 생성된 딜과 Won된 딜을 각각 검색해서,
주간 영업 리포트 형식으로 정리해줘.
```

### 고객 인수인계

```
"이영희" 담당 고객과 딜을 모두 검색해서 목록 정리하고,
각 딜의 현재 상태와 최근 활동을 요약해줘.
```

## 프롬프트 팁

### 구체적으로 요청하세요

```
x  "딜 보여줘"
o  "이번 달 생성된 딜 중 금액 1000만원 이상인 것 검색해줘"
```

### 분석을 요청하세요

AI가 데이터를 조회한 후 직접 분석·요약해 줍니다.

```
"지난 분기 Lost된 딜들을 검색해서, 공통 패턴이 있는지 분석해줘."
```

### 링크를 요청하세요

세일즈맵 웹에서 직접 확인하고 싶을 때 URL을 받을 수 있습니다.

```
"이 딜의 세일즈맵 링크도 알려줘"
```

### 여러 작업을 한 번에 요청할 수 있습니다

```
"이 고객 정보 업데이트하고, 노트도 남기고, 연결된 딜 현황도 알려줘."
```

## 제한사항

| 항목 | 설명 |
|------|------|
| 검색 필터 | 최대 3개 그룹, 그룹당 3개 조건 |
| 일괄 조회 | 한 번에 최대 20개 레코드 |
| 삭제 | 딜/리드만 삭제 가능 (고객·회사는 세일즈맵 웹에서 직접 삭제) |
| 파일 첨부 | 파일 업로드/다운로드 미지원 |
| 이메일 | 이메일 발송 미지원. 수신 이메일은 제목만 조회 가능 (본문 미지원) |
| 활동 생성 | 노트만 생성 가능. TODO, 미팅, SMS 등은 생성 미지원 |
| 검색 정렬 | 서버 정렬 미지원 — 결과 내에서 AI가 정리해 줍니다 |
