## 1. [Bug] singleSelect 필드 create/update 시 값 미저장

### 1.1 현상
MCP API를 통해 Lead 또는 Organization의 singleSelect 타입 필드에 값을 넣어 create 또는 update 호출 시, API 응답은 정상(200)이지만 실제 레코드에 값이 저장되지 않음.

### 1.2 재현 환경
| 항목 | 값 |
|------|-----|
| 워크스페이스 | Polaris3D (token: 82e203b9-...) |
| API | salesmap_create_record / salesmap_update_record |
| 대상 오브젝트 | Lead, Organization |
| 재현율 | 100% (singleSelect 타입 필드 전부) |

### 1.3 필드 타입별 동작 비교

| 필드 타입 | 예시 필드 | 결과 | 비고 |
|-----------|-----------|------|------|
| string | 문의 내용, 매장/공장 규모 | OK | 정상 저장 |
| boolean | 의사결정자 접촉 여부 | OK | 정상 저장 |
| number | 금액 | OK | 정상 저장 |
| multiSelect | 관심 솔루션, 리드 소스 | OK | stringValueList 사용 |
| singleSelect | 업종, 리드 등급, 도입 희망 시기, 예산 확보 여부, 정부지원사업 해당 | **FAIL** | 에러 없이 무시됨 |

### 1.4 재현 호출 예시

**Case 1: create 시 singleSelect 포함**
```
salesmap_create_record(
  type: "lead",
  name: "교촌치킨 강남직영점 - 서빙로봇 도입",
  fieldList: [
    { name: "리드 등급", stringValue: "🟢" },        // ❌ 미저장
    { name: "도입 희망 시기", stringValue: "1개월 내" }, // ❌ 미저장
    { name: "문의 내용", stringValue: "테스트" },       // ✅ 저장됨
  ]
)
```

**Case 2: update 시 singleSelect 단독**
```
salesmap_update_record(
  id: "019d0508-d176-...",
  type: "lead",
  fieldList: [
    { name: "리드 등급", stringValue: "🟢" }  // ❌ 미저장
  ]
)
// 응답: 200 OK, updatedAt 변경 안 됨
```

**Case 3: update 시 stringValueList 시도**
```
salesmap_update_record(
  fieldList: [
    { name: "리드 등급", stringValueList: ["🟢"] }  // ❌ 에러 반환
  ]
)
// 에러: "lead 리드 등급에 stringValue가 없습니다"
```

### 1.5 기대 동작
singleSelect 필드에 stringValue로 옵션 값(label)을 전달하면 해당 옵션으로 저장되어야 함. 또는 옵션 ID를 받는 별도 파라미터(예: optionId)를 지원해야 함.

### 1.6 영향 범위
AI Agent가 CRM 데이터를 자동 생성/업데이트하는 시나리오에서 singleSelect 필드를 프로그래밍 방식으로 설정할 수 없음. 데모 워크스페이스 구축, 자동화 워크플로우, 웹폼 후속 처리 등 대부분의 MCP 활용 시나리오에 영향.

## 2. [VOC] Tool 구조 개선 요청 — 기능 중심 재구성

### 2.1 배경
Claude 등 AI Agent는 한 턴에 호출할 수 있는 tool 수에 제한이 있음 (실측 약 25~30회). 현재 tool 구조는 레코드 단위로 설계되어 대량 데이터 작업 시 턴이 자주 끊기고 사용자가 '계속'을 반복 입력해야 함.

### 2.2 현재 문제 — 20개 고객사 데모 데이터 생성 사례

| 작업 | 레코드 수 | API 호출 수 | 필요 턴 수 |
|------|-----------|-------------|------------|
| Organization 생성 | 15개 | 15회 | 1턴 |
| People 생성 (org 연결) | 15개 | 15회 | 1턴 |
| Lead 생성 (people+org 연결) | 15개 | 15회 | 1턴 |
| Deal 생성 | 13개 | 13회 | 1턴 |
| Lead 필드 보강 (update) | 15개 | 15회 | 1턴 |
| Org 필드 보강 (update) | 15개 | 15회 | 1턴 |
| 스키마 확인 + 파이프라인 조회 | - | ~10회 | - |
| **합계** | - | **~98회** | **4~5턴** |

### 2.3 개선 제안

#### 제안 A: Batch Create/Update API
```
salesmap_batch_create_records(
  type: "organization",
  records: [
    { name: "교촌치킨 강남직영점", fieldList: [...] },
    { name: "현대백화점 압구정본점", fieldList: [...] },
    // ... 최대 20개
  ]
)
```

기대 효과:
| 작업 | 현재 (개별 호출) | 개선 후 (배치) |
|------|-----------------|---------------|
| Org 15개 생성 | 15회 호출 | 1회 호출 |
| People 15개 생성 | 15회 호출 | 1회 호출 |
| Lead 15개 생성 | 15회 호출 | 1회 호출 |
| Deal 13개 생성 | 13회 호출 | 1회 호출 |
| **합계** | **58회** | **4회** |

#### 제안 B: Batch Update API
```
salesmap_batch_update_records(
  type: "lead",
  records: [
    { id: "019d0508-...", fieldList: [{ name: "리드 등급", stringValue: "🟢" }] },
    { id: "019d0509-...", fieldList: [{ name: "리드 등급", stringValue: "🟡" }] },
    // ... 최대 20개
  ]
)
```

#### 제안 C: Composite API (Create + Link)
```
salesmap_create_linked_records(
  organization: { name: "교촌치킨", fieldList: [...] },
  people: { name: "김태현", fieldList: [...] },
  lead: { name: "서빙로봇 도입", fieldList: [...], pipelineId: "...", pipelineStageId: "..." }
)
// 한 번의 호출로 org + people + lead 생성 및 자동 연결
```

### 2.4 우선순위 제안

| 우선순위 | 제안 | 효과 | 난이도 |
|---------|------|------|--------|
| **P0** | singleSelect 버그 수정 | 기본 기능 정상화 | 낮음 (버그 수정) |
| **P1** | Batch Create/Update | 호출 수 93% 감소 | 중간 |
| **P2** | Composite API | UX 획기적 개선 | 높음 |
